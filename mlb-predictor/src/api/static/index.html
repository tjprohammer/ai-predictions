Board Controls

Load a slate or rerun the scoring pass
Refresh Everything
Full pipeline — schedule, lineups, markets, aggregates, features, predictions, and published board. Mid-slate, this can refresh odds and rescored edges for games that have not started yet; use only when you want the whole board rebuilt. For long runs, start the API with start-app-stable (no --reload) so the job is not interrupted.
Prepare Slate
Loads schedule & starters, seeds the CSV templates, imports current template state, and republishes the board.
Update Lineups & Markets
Re-reads the lineup and market CSVs, then rebuilds the board so manual edits show up immediately. Calls the Odds API for markets.
Update Lineups
Lineup CSVs and player status only — no Odds API. Use while lineups are still in flux so you are not burning market quota on every refresh.
Update Markets
Odds and player-prop market ingest only — does not re-import lineup CSVs. Use when books move lines and you do not need another lineup pass.
Rebuild Predictions
Re-runs feature builds and prediction scoring for the selected date without re-fetching external data.
Retrain Models
Re-trains all four prediction models (totals, first-5, hits, strikeouts) on current data, then re-scores predictions for the selected date.
Refresh Daily Results
Pulls box scores, batting logs, and observed weather, then republishes surfaces — without re-importing markets or rebuilding prediction features. Prefer this while games are in progress (staggered starts). Run Grade Predictions after games go final.
Grade Predictions
Grades picks and refreshes scorecards using only games that are already final.
